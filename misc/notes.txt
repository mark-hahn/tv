
Client requirements: per-show Notes persistence

- UI: In the Series pane (both simple and non-simple modes), show a multiline notes input box directly below the existing Series infobox, with the same width as the infobox.

- RPC functions (tv-series-srvr):
	- `getNote(showName)`
		- Input: `showName` string
		- Output: note text (string) OR an object containing a string field like `{ noteText: "..." }`.
	- `saveNote(showName, noteText)`
		- Input: `showName` string and `noteText` string
		- Output: `ok` or an error (same pattern as other RPC calls).

- Persistence behavior:
	- When the Series pane loads a show, client calls `getNote(showName)` and populates the notes box.
	- When the notes value is committed (client uses blur), client calls `saveNote(showName, noteText)`.

- Server storage:
	- The tv-series-srvr should persist notes to disk so they survive app restarts.
	- One note per show name.
